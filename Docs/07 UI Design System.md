# UI Design System  
## Product  
RUMA Events  
## Design Direction  
The interface should feel festive, modern, simple, and premium.  
The visual style should be inspired by Onam and Kerala, but should avoid looking crowded, overly decorative, or poster-like.  
The design should feel like:  
Modern Kerala × Community Celebration × Clean Digital Ticketing  
  
## Core Principles  
## 1. Mobile First  
Most users will open the platform from WhatsApp on mobile.  
Design for mobile first, then scale to desktop.  
## 2. Festive but Minimal  
Use Onam-inspired colors and patterns subtly.  
Avoid excessive flowers, gradients, icons, or decorative clutter.  
## 3. Clear and Trustworthy  
Payment, ticket status, and approval status should always be clear.  
## 4. Family Friendly  
The experience should feel easy for families, elders, and non-technical users.

* Inter  
* Geist  
* Manrope

Page Title 32px / 40px  
Section Title 24px / 32px

Body 16px / 24px

## Layout  
## Mobile  
* Single column  
* 16px page padding  
* Sticky bottom CTA where useful  
* Large tap targets  
## Desktop  
* Max content width: 1120px  
* Center-aligned content  
* Two-column layouts only when helpful

* 4px  
* 8px  
* 12px  
* 16px  
* 24px  
* 32px  
* 48px  
* 64px  
  
## Border Radius  
Small 8px  
Medium 12px  
Large 20px  
Ticket Cards 24px  
  
## Buttons  
## Primary Button  
Background: Kerala Green Text: White Radius: 12px Height: 48px minimum  
Used for:  
* Register Now  
* Submit Registration  
* Approve Payment  
* Download Ticket  
## Secondary Button  
Background: Soft Cream Text: Kerala Green Border: Kerala Green 1px  
Used for:  
* Back  
* View Details  
* Edit  
## Danger Button  
Background: Light Red Text: Dark Red  
Used for:  
* Reject Payment  
* Cancel Ticket  
  
## Form Fields  
Fields should be large, clean, and easy to tap.  
Height: 48px minimum Radius: 12px Border: #D0D5DD  
Required fields should be clearly marked.  
Validation messages should appear directly below the field.  
  
## Ticket Selector  
Ticket quantity controls should use stepper buttons.  
Example:  
Adults 13+ [-] 2 [+]  
Children 5–12 [-] 1 [+]  
Children 5–12 [-] 1 [+]  
Children Below 5 [-] 0 [+]  
Each row should show:  
* Ticket type  
* Age rule  
* Price  
* Quantity selector  
  
## Payment Summary Card  
Should show:  
* Adults count and amount  
* Children count and amount  
* Free children count  
* Lucky draw coupons  
* Total payable  
The total payable should be visually prominent.  
  
## Ticket Design  
Each ticket should feel like a premium digital pass.  
Ticket should include:  
* RUMA Onam title  
* Ticket number  
* Booking reference  
* Ticket type  
* Name  
* Flat number  
* QR code  
* Status  
Visual style:  
* Ivory/cream background  
* Green header band  
* Gold decorative accent  
* Subtle pookalam pattern watermark  
* Rounded corners  
* Clear QR area  
  
## QR Status States  
## Valid  
Color: Green Message: Valid Ticket  
## Already Checked In  
Color: Amber Message: Already Checked In  
## Invalid  
Color: Red Message: Invalid Ticket  
  
## Admin Dashboard Cards  
Cards should show:  
* Total registrations  
* Approved registrations  
* Pending approvals  
* Total revenue  
* Adults count  
* Children count  
* Checked-in count  
Use simple cards with large numbers and short labels.  
  
## Empty States  
Empty states should be friendly and useful.  
Example:  
No registrations yet Once residents start registering, submissions will appear here.  
  
## Loading States  
Use skeleton loaders for:  
* Dashboard cards  
* Registration list  
* Ticket details  
* Payment screenshots  
  
## Error States  
Error messages should be simple and human.  
Example:  
We could not upload your payment screenshot. Please try again.  
Avoid technical errors.  
  
## Onam Visual Elements  
Use subtly:  
* Pookalam patterns  
* Banana leaf shapes  
* Coconut tree silhouettes  
* Gold line accents  
* Kerala kasavu-inspired border lines  
Avoid:  
* Too many flowers  
* Heavy gradients  
* Low-quality clipart  
* Overdecorated backgrounds  
* Festival poster style  
  
## Theme Customization  
The platform should support event-level theme settings.  
Each event can define:  
* Primary color  
* Accent color  
* Background color  
* Event logo  
* Hero image  
* Ticket background pattern  
This allows future events like Vishu, Christmas, New Year, or Sports Day to use the same platform with different visual themes.  
  
## Accessibility  
Minimum requirements:  
* Good color contrast  
* Keyboard navigable forms  
* Clear focus states  
* Large touch targets  
* Descriptive button labels  
* QR fallback ticket number  
  
## Overall UI Quality Bar  
The product should feel:  
* Clean  
* Warm  
* Premium  
* Trustworthy  
* Community-friendly  
* Festive without clutter  
The UI should not feel like a Google Form replacement.  
It should feel like a proper RUMA-branded digital event experience.
