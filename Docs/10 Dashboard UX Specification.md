# Dashboard UX Specification  
  
## Product  
  
RUMA OS  
  
## Purpose  
  
The dashboard is the operational hub for managing the RUMA community.  
  
It enables volunteers and committee members to manage events, registrations, tickets, attendance, content, and community activities from a single interface.  
  
The dashboard should feel approachable, modern, and simple enough for non-technical volunteers.  
  
---  
  
# UX Principles  
  
## Community First  
  
Show people and events before data.  
  
Avoid corporate admin experiences.  
  
---  
  
## Action Oriented  
  
Users should immediately understand:  
  
- What needs attention  
- What happened recently  
- What action should be taken next  
  
---  
  
## Minimal Complexity  
  
Hide complexity behind progressive disclosure.  
  
Show only what is relevant.  
  
---  
  
## Mobile Friendly  
  
Committee members should be able to approve registrations directly from WhatsApp links on mobile.

# Success Metric  
  
The dashboard should allow a volunteer to:  
  
Create an event  
  
Approve a registration  
  
Generate tickets  
  
Scan attendees  
  
Upload photos  
  
View reports  
  
without requiring training.
